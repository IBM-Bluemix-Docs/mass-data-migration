# IBM Cloud 大量マイグレーション・デバイスへのデータのインポート
{: #userguide}

{{site.data.keyword.cloud}} 大量マイグレーション・デバイスは、マウント可能なネットワーク・ファイル・システム (NFS) 共有または FileNet Content Federation Services (CFS) 共有を提供できるポータブル・ストレージ・デバイスです。 このデバイスは、Web ブラウザー・インターフェースで管理します。 このデバイスがお客様のデータ・センターに配送されたら、オンサイトでデータをロードし、{{site.data.keyword.BluSoftlayer_full}} データ・センターに送り返してください。その後、弊社がお客様の {{site.data.keyword.cos_full}} のアカウントにデータをロードします。


## デバイスへの電源供給

このデバイスは、C13-US の電源コード ([https://en.wikipedia.org/wiki/IEC_60320 ![外部リンク・アイコン](../../icons/launch-glyph.svg "外部リンク・アイコン")](https://en.wikipedia.org/wiki/IEC_60320){:new_window}) が同梱されて送付されます。このデバイスを米国以外の場所で使用する場合は、電源アダプターが必要になる場合があります。

このデバイスは、標準的なすべての電力範囲に対応しています。 <br/>
![電力範囲](/images/PowerRating.png)

このデバイスの電源を入れるには、以下の手順に従ってください。
1. 電源プラグの横の「Mains Switch」をオンにします。 <br/>
   ![Mains Switch](/images/MDMSPowerOnOff.png)

2. 接続リンク LED の横にある「System On / Off」ボタンを使用します。
   ![System On / Off](/images/MDMSSystemOnOff.png)

このデバイスは、System ID が LED 画面に表示されると電源オン状態になります。


## イーサネット接続の構成

2 つのイーサネット接続を確立する必要があります。 1 つはブラウザーでデバイスを管理するための接続、もう 1 つはソース・データが置かれているサブネットと同じサブネットでデータを移動するための接続です。

{{site.data.keyword.cloud}} では、2 つのモデルの MDMS デバイスを提供しています。 1 つのモデルでは、RJ45 接続だけをサポートします。 もう 1 つのモデルでは、SFP+ 銅線と RJ45 をサポートします。 MDMS デバイスのモデルに応じて、適切な手順を実行します。

デフォルトでは、10 GbE ポートでジャンボ・フレームが有効です。 この設定は、UI の「ネットワーク・ポートの変更 (Modify Network Port)」オプションを使用して変更できます。
{:tip}

### RJ45 だけの構成

![RJ45](/images/RJ45PortZoom.png)

デバイスのポートは RJ45 に対応していて、CAT6A ケーブルが同梱されています。 RJ45 から変換する場合の SFP+ 銅線用のアダプターも用意されています。 このアダプターは、すべてのメーカーのスイッチに対応しています。 配送用コンテナーのふたの裏側にあるポケットにアダプターが入っています。

- Eth1 (`1GbE-B`) は、通常はデバイス管理用に使用されるので、IP アドレスの構成でゲートウェイを指定する必要があります。 この構成情報は、デバイスの電源を入れた後に LCD 画面に表示されます (『IP アドレスの構成』セクションを参照)。 このポートは、Web ベースの UI をデータ・サブネット外で使用できるようにするために使用します。

- Eth3 (`10GbE-B`) はデータ転送用に使用され、デバイスの管理にも使用できます。 この接続は、ソース・データと同じサブネットで行う必要がありますが、必要であればサーバーに直接接続することも可能です。


### SFP+ 銅線および RJ45 の構成

![SFP+ 銅線](/images/sfp-ports-sized-port5.png)

デバイスのポートは SFP+ 銅線および RJ45 に対応しています。 CAT6A ケーブルと SFP+ 銅線ケーブルの両方が同梱されています。

- Eth5 (`10Gb SFP+ (5)`) は、通常はデータ転送用に使用されますが、デバイスの管理にも使用できます。 このポートは 10 GbE の速度でのみ動作します。

- Eth2 (`10-GbE (2)`) は、通常はデバイス管理用に使用されますが、データ転送にも使用できます。 このポートは、1 GbE または 10 GbE のいずれかの速度で実行できます。


データ転送用の接続は、ソース・データと同じサブネットで行う必要がありますが、サーバーに直接接続することも可能です。

IP 設定は、デバイスの電源を入れた後に LCD 画面で表示および管理できます (『IP アドレスの構成』セクションを参照)。

Web ブラウザーで 10 GbE ポートの IP にアクセスできる場合は、両方のポートを構成および使用する必要はありません。
{:note}


## データのロード

1.	このデバイスは、IP アドレス、ユーザー名、ロックされたストレージ・プール、NFS 共有が事前に構成された状態で届けられます。 ユーザーのパスワードとストレージ・プールのパスワードは、E メールで別途通知されます。

2.	デバイスの最適な設置場所を決めます。 デバイスが電源とイーサネット接続の両方に届き、移動距離が最小限で済む場所を選択してください。

3.	接続のためにデバイスを配置します。 ポータブル・ケースからデバイスを取り出す必要はありません。 移送ケースに入れたままでも使用できます。 デバイスが室温の状態で、結露がないことを確認してください。 ケースのふたの裏側にある付属の電源ケーブルを使用してコンセントに接続し、デバイスの電源を入れます。<br/>

    2 つの電源スイッチに注目してください。
    {:note}
    ![電源スイッチ](/images/MDMSPowerSwitch.png)

4. デバイスをネットワークに接続します。
    - RJ45 の接続
      1. ケースのふたから CAT6A ケーブルを取り外して、Eth3 (10 GbE-B) ポートに接続します。
         ![MDMS デバイスの各ポート](/images/MDMSNewEth1and3.png)
      2. 付属の CAT6A を SFP+ アダプターに接続し、それを 10 Gb スイッチに接続します。
      3. ブラウザーに `HTTPS://'Your-Eth3-IPAddress'` と入力して、Eth3 に構成された IP アドレスにアクセスできる場合は、次の手順に進みます。 アクセスできない場合は、Eth1 (`1GbE-B`) ポートに接続します。<br/>

         Eth3 または Eth1 の IP 設定を変更する必要がある場合は、[IP アドレスの構成](#configuring-ip-addresses)セクションを参照してください。
         {:tip}

    - SFP+ 銅線の接続
      1. ケースのふたから SFP+ 銅線ケーブルを取り外して、Eth5 10 GbE (5) に接続します
         ![MDMS デバイスの各ポート](/images/sfp-ports-sized-ports-labeled.png)
      2. この SFP+ 銅線ケーブルをお客様の 10 Gb スイッチに接続します。
      3. ブラウザーに `HTTPS://'Your-Eth5-IPAddress'` と入力して、Eth5 に構成された IP アドレスにアクセスできる場合は、次の手順に進みます。アクセスできない場合は、Eth2 (`10GbE-B` または `1GbE-B`) ポートを接続します。

         Eth5 または Eth2 の IP 設定を変更する必要がある場合は、[IP アドレスの構成](#configuring-ip-addresses)セクションを参照してください。
         {:tip}

5. ブラウザーを開き、`HTTPS://Your-Eth1-IPAddress` と入力します。 `Your-Eth1-IPAddress` をご使用のネットワーク構成の Eth1 に置き換えます。 証明書の例外は、そのまま受け入れます。

6. 通知されたユーザー名とパスワードを使用してログインします。<br/>
    ![ログイン・ページ](/images/login.png)

7. ワークフロー・ウィザードに、特定の項目へのアクセス用のアイコンが表示されます。通常は、左から右の順に使用します。<br/>
    ![ワークフロー・アイコン](/images/workflow.png)

    このワークフローは、インターフェースの左上領域にある**「Workflow Manager」**を使用して再度開くことができます。
    {:tip}

8.	事前構成されたストレージ・プールをアクティブにします。
    - **「ストレージ・プールのロックを解除して開始 (Unlock and Start Storage Pool)」**をクリックします。
    - ストレージ・プール・パスフレーズを入力して、**「OK」**をクリックします。
      ![ストレージ・プールのアクティブ化](/images/Unlock.png)

9. この共有を行うときに、デフォルトでは NFS プロトコルと SMB プロトコルの両方がアクセス制限なしで有効になります。 この共有 (NFS または SMB) へのアクセスを制限する場合は、共有名を右クリックし、該当するメニュー項目を選択してください。<br/>
   ![共有アクセスの制限](/images/ShareAccessControl.png)

10. ストレージ・プールが有効になると、NFS 共有のマウントが可能になります。 ワークフローで**「ネットワーク共有の表示 (View Network Shares)」**をクリックすると、ネットワーク共有を示すビューが表示されます。 ワークフローを閉じて、共有を右クリックし、マウント・コマンドを選択すると、共有名とマウント情報が表示されます。 ソース・サーバーに共有をマウントします。 必ず 10 GB リンクの IP アドレスを指定してください。
    ![共有のマウント](/images/MountCommand.png)

11. データを NFS 共有にコピーします。 ワークフローで**「ネットワーク・アクティビティーの表示 (View Network Activity)」**をクリックすると、データが 10 GB リンクでデバイスに転送されているインバウンド・イーサネット・ロードが表示されます。
    ![アクティビティーの表示](/images/SystemNetworkPerf.png)

12. ワークフローで**「ストレージ・プールの表示 (View Storage pool)」**をクリックすると、デバイスのストレージの使用状況と IOPS をモニターできます。
    ![ストレージ・プールの表示](/images/SystemStoragePoolPerf.png)

13.	ロードが完了したら、システムの電源を正常にオフにすることができます。 ワークフローで**「アプライアンスのシャットダウン... (Shutdown Appliance...)」**をクリックします。
    ![アプライアンスのシャットダウン](/images/SystemShutdown.png)

14.	デバイスの接続を切断し、電源ケーブル、イーサネット・ケーブル、SFP+ アダプターをふたの裏側の保管場所に戻します。

16.	付属されていた配送ラベルを貼り付け、配送業者に連絡し、デバイスをデータ・センターに返却します。


## IP アドレスの構成

デバイスの LCD 画面を使用して、イーサネット・ポートの IP アドレスを構成できます。 LCD パネル内でのカーソルの移動は、**Up**、**Down**、**Back/ESC**、**Forward/ENTER** ボタンを使用して行います。 **Enter** を押すとメニューが表示され、**Exit** を押すと終了します。

IP アドレスやサブネット・マスクの編集時は、**Enter** で 1 文字ずつ前に移動し、**Exit** で 1 文字ずつ後に移動します。

選択した場所の数字の切り替えには、**上**と**下**のボタンを使用します。

**Exit** を使用すると、前のメニューに戻ります。

**「更新... (Update...)」**に移動して **Enter** を押すと、設定を保存できます。

  ![LCD ディスプレイ](/images/MDMSLCD.png)
